At this time we created an application with a MySQL database to store tasks. Unfortunatly we use an ephemeral mysql storage. Indeed if we stop our mysql pod, we will loose all our database content.

To confirm it, stop your mysql database pod :

oc get pods | grep Running # get the mysql running pod name
 oc stop pod workshop-db-Y-YYYYYY

To avoid this we need persistent storage.

First get the name of the volume:

oc volume dc/workshop-db --list

Output::

deploymentconfigs/workshop-db
  empty directory as workshop-db-volume-1
    mounted at /var/lib/mysql/data
  empty directory as workshop-db-volume-2
    mounted at /run

The output show that this pod use 2 volumes and the directory /var/lib/mysql/data is on workshop-db-volume-1.

Then make your data persistent. In the command bellow, replace workshop-db-volume-X by your own value. After launching the command, look at what hapen in the overview window of the console.

oc volume dc/workshop-db --add --name=workshop-db-volume-X -t pvc --claim-name=pvc1 --claim-size=3G --claim-mode=ReadWriteMany --overwrite

In the web console, you can see a new pod appearing and then the old one disapear: a "build config" object is created wich create an image. Then a 'deploy config' customyze your app. Finally the resulting pod is launched and appear running. And all those steps are done inside diffrent containers!

Now if you launch this command again:

oc volume dc/workshop-db --list

"empty directory" (ephemeral storage in openshift) has been replaced by the physical volume claim :

  empty directory as workshop-db-volume-1
    pvc/pvc1 (allocated 3GiB) as mysqldb-jlb-volume-1
  empty directory as workshop-db-volume-2
    mounted at /run

You can also check the persistence by login into your pod with this command:

oc exec workshop-db-Y-YYYYY -it bash
